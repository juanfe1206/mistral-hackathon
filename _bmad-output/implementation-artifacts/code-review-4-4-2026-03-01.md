## 🟡 HIGH ISSUES

### 1. "Preserve triage context" is only encoded in URL, not restored in triage state

**Finding:** The story marks triage context preservation complete, but the triage page does not read query params to initialize filters/sort when navigating back from lead detail.

**Evidence:**
- Story task marked complete: preserve triage context (`_bmad-output/implementation-artifacts/4-4-align-triage-and-at-risk-flows-to-spec.md`, Task 2).
- Lead detail back link sends query params (`sort`, `priority`, `lifecycle`, `source`) to `/triage` (`src/app/(dashboard)/lead/[id]/page.tsx`).
- Triage page initializes `filters` to `{}` and `sort` to `"priority_desc"` and never reads `searchParams` (`src/app/(dashboard)/triage/page.tsx`).

**Impact:** Back-and-forth navigation does not actually restore prior triage filter/sort UI state, so AC #2 ("interaction cost") is only partially implemented.

**Recommendation:** Parse query params on triage load (and on navigation updates) to seed `filters`/`sort`, with validation against allowed enum values.

---

## 🟠 MEDIUM ISSUES

### 2. Test coverage validates link generation, but not context restoration behavior

**Finding:** Integration tests check that triage links include query params, but they do not verify that triage restores those params after returning from lead detail.

**Evidence:**
- Current test asserts `href` contains `from=triage` and `sort=priority_desc` (`tests/integration/triage-at-risk-flow-alignment.test.tsx`).
- No integration test covers a full round trip (triage -> lead detail -> back to triage with filters/sort restored).

**Recommendation:** Add a test that navigates through lead detail and asserts restored filter/sort state in `QueueFilterBar` on return.

---

### 3. Story File List does not match staged source changes in the branch

**Finding:** The story file list is incomplete relative to current staged changes.

**Evidence:**
- Story File List includes nine files only (`_bmad-output/implementation-artifacts/4-4-align-triage-and-at-risk-flows-to-spec.md`).
- Staged changes also include source/test files not listed there (for example `src/components/SLASafetyIndicator.tsx`, `src/server/services/sla-service.ts`, `tests/api/leads-detail.test.ts`, and governance files).

**Impact:** Review traceability is reduced and it is harder to isolate story-specific scope in a mixed branch.

**Recommendation:** Either isolate 4.4 changes in a clean commit/snapshot or update story documentation to explicitly separate in-scope and out-of-scope concurrent edits.
