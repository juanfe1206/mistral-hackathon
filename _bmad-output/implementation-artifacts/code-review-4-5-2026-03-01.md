## 🟡 HIGH ISSUES

### 1. KPI retry control violates 44px touch-target requirement

**Finding:** The KPI unavailable-state retry action is explicitly sized below the story's 44px mobile touch-target minimum.

**Evidence:**
- Story AC #1 requires touch targets to be at least 44px on mobile.
- `src/components/KPISummaryPanel.tsx:84` sets `minHeight: 36` for the Retry button.

**Impact:** Mobile accessibility target size is not met for a critical recovery action.

**Recommendation:** Increase the Retry button target to at least 44px (`minHeight: 44`) and keep it consistent with other action controls.

---

### 2. Settings error-state retry action is below required touch-target size

**Finding:** The error fallback in Settings adds an interactive retry control without enforcing the 44px minimum.

**Evidence:**
- Story AC #1 requires 44px minimum touch targets.
- `src/app/(dashboard)/settings/page.tsx:59` renders an outlined MUI `Button` without `minHeight`; default sizing is below 44px.

**Impact:** Error recovery path can fail accessibility target-size expectations on mobile.

**Recommendation:** Set explicit mobile-safe target size (`sx={{ minHeight: 44 }}`) and keep consistency across recovery actions.
