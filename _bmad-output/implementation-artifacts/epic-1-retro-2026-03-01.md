## Challenges and Growth Areas

1. **pnpm setup friction** — Initial setup was hard (especially on Windows); once configured, flow was smooth.
2. **Webhook payload complexity** — Nested `entry[].changes[].value` and multi-message handling required iteration.
3. **NFRs caught in review** — NFR11 (ingestion failures visible) and NFR1 (2s refresh) surfaced in code review, not first implementation.
4. **Atomicity and validation** — Lead+interaction transaction and timeline API validation (invalid params → 400) were review findings.
5. **Test execution clarity** — Unclear whether AI had run tests; confirmed 16 tests pass via `pnpm test`.

## Key Insights

1. **Supabase as MVP DB** — Free tier and ease of use justified the choice; Prisma abstraction keeps options open.
2. **Code review adds value** — Consistently caught NFR gaps, atomicity, and boundary validation.
3. **Include NFRs in story prep** — Dev notes should list relevant NFRs before implementation to reduce rework.
4. **Run tests before marking done** — Establish habit: `pnpm test` must pass before story completion.
